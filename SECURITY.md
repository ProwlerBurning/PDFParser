# Security Policy

This project is **local-first and privacy-sensitive**. It parses financial
statements on your own machine. **No real financial statements are included in
this repository.**

## Supported versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | ✅        |
| < 0.1.0 | ❌        |

## Reporting a vulnerability

Please report security issues **privately**, not in public issues or pull
requests. Use GitHub's private vulnerability reporting for this repository, or
maintainer contact once configured. Include a clear description and reproduction
steps using **synthetic or redacted** data only.

## Do NOT attach real or sensitive data

When reporting any issue (security or otherwise), **never attach or paste**:

- real PDFs or real bank / e-wallet statements,
- account numbers, wallet IDs, or card numbers,
- authentication tokens, API keys, or credentials,
- private transaction data.

Use synthetic or fully redacted examples instead. Logs must be scrubbed of any
personal identifiers before sharing.

## Suspected data exposure

If you believe real personal or financial data has been exposed (for example,
accidentally committed or attached), report it **privately** using the channels
above so it can be addressed before any public discussion.
